# 🧠 brain-manager: Jekyll 지식 자동화 스킬

## 📌 스킬 개요
사용자가 입력한 지식을 바탕으로 Jekyll 호환 개별 포스트를 생성하고, 해당 카테고리의 메인 인덱스(index.md)를 자동으로 갱신하거나 구조화합니다. 마크다운 문서 작성 시 표준 문법 가이드라인을 적용하여 렌더링 최적화와 가독성 통일을 목적으로 합니다.

## 🎯 트리거 명령어
> **"지식화 하여라"** 또는 **"지식화 해줘"**

## 📝 [Rule 1] 개별 지식 페이지 생성 (Individual Page)
1. **Jekyll Frontmatter**
   - **tags**: 본문에서 핵심 단어를 추출하되, 영문 지상주의를 탈피합니다. 한글 용어가 더 직관적이라면 한글 키워드를 우선 사용합니다.
   - 예: `tags: [가비지컬렉션, 메모리관리, C#]` (O) / `tags: [GarbageCollection, MemoryManagement, CSharp]` (X - 너무 길고 복잡함)

2. **본문 구조**
   - **Abstract**: 시작 직후 `>` 인용구로 2~3줄 요약.
   - **Navigation**: 최하단에 `[🔙 뒤로 가기](./index.md)` 링크 필수.

## 📝 [Rule 2] 메인 인덱스 업데이트 (index.md)
1. **인덱스 테이블 구성**
   - **형식**: `| 📑 핵심 아티클 (Title) | 🏷️ 분류 태그 (Tags) |`
   - **내용**: `[[카테고리] {{Dynamic_Emoji}} {{제목}}](./파일명.md)`

2. **키워드(태그) 추출 규칙 (중요!)**
   - **한글 우선 원칙**: 한국어 기술 블로그로서 읽기 편한 단어를 선택합니다.
   - **자연스러운 혼용**: #인터페이스, #추상화처럼 한글이 편한 건 한글로, #API, #JSON처럼 영문 약어가 표준인 건 영문으로 표기합니다.
   - **형식**: 백틱과 샵 사용 (예: `` ` #메모리 ` `` `` ` #GC ` ``)

# 🛠️ 문법 (Jekyll 지식 문서 표준 가이드라인)

## 표 (Table)
- **전후 여백**: 표의 시작 직전과 종료 직후에는 반드시 빈 줄(Empty Line)을 한 줄 이상 삽입합니다.
- **파이프 정렬**: 모든 행의 시작과 끝을 `|` 기호로 닫아 구조를 명확히 합니다.
- **콘텐츠 보호**: 셀 내부에서는 파이프(`|`) 기호 사용을 지양하고, 필요시 `\|`로 이스케이프 합니다.
- **가독성**: 제목 링크는 `**[제목](./경로)**`와 같이 볼드 처리하여 시각적 위계를 둡니다.

## 코드 블록 (Code Block)
- **언어 명시**: ` ``` ` 뒤에 반드시 해당 코드의 언어(csharp, bash, yaml 등)를 명시합니다.
- **주석**: 코드 설명 주석은 한국어를 우선 사용하며, 핵심 로직에 상세히 기술합니다.
- **여백**: 코드 블록 위아래로 빈 줄을 두어 본문과 구분합니다.

## 콜아웃 (Callouts)
- **GitHub 스타일**: `> [!NOTE]`, `> [!TIP]`, `> [!WARNING]` 형식을 적극 활용합니다.
- **용도 구분**: 단순 정보는 NOTE, 유용한 팁은 TIP, 주의사항은 WARNING으로 분류하여 독자의 주의를 환기합니다.

## 키워드 및 태그 (Keywords & Tags)
- **언어 선택**: 억지 영문 변환보다는 한국어 기술 용어(예: #의존성주입)를 우선 사용합니다.
- **혼용 원칙**: 표준 약어(예: #API, #GC)는 영문 대문자로 표기하여 전문성을 유지합니다.
- **강조**: 태그는 반드시 백틱과 샵을 조합( `` ` #태그 ` `` )하여 인라인 코드 형식으로 강조합니다.

## 이모지 (Dynamic Emoji)
- **맥락 반영**: 단순 매핑이 아닌, 문서의 성격(이론 🧠, 팁 ✨, 해결 🛠️, 성능 🚀)에 맞춰 AI가 판단하여 삽입합니다.
- **위치**: 문서의 메인 타이틀(H1)과 인덱스 테이블의 제목 앞에 배치합니다.
- **언어별 상징**: C#(🎯), Python(🐍), JS(💛), DB(🗄️) 등 (문맥에 따라 조합 가능)

## 🤖 [AI 에이전트 지시사항]
1. 사용자가 지식을 입력하면 가장 먼저 **# 문법** 섹션의 규칙을 로드하라.
2. **언어 선택의 유연성**: 키워드 추출 시 "개발자가 구글에 검색할 법한 단어" 혹은 "본문에서 주로 사용한 단어"를 기준으로 한/영을 선택하라. 억지로 영단어로 치환하지 마라.
3. **맥락 기반 이모지**: 단순히 기술명에 매몰되지 말고, 이 글이 '삽질 기록'인지 '개념 정리'인지에 따라 이모지를 다채롭게 사용하라.
4. **Jekyll 규격**: Frontmatter의 형식(날짜, 레이아웃 등)이 깨지지 않도록 엄격히 관리하라.
5. **가독성**: 테이블 내의 제목 링크가 본문의 실제 파일명과 일치하는지 확인하라.
