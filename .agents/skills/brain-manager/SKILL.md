# 🧠 brain-manager: Jekyll 지식 자동화 스킬

## 📌 스킬 개요
사용자가 입력한 지식을 바탕으로 **Jekyll 호환 포스트**를 생성하고, 카테고리별 **인덱스** 및 **메인 대시보드**를 구조화합니다. 모든 출력물은 GitHub Pages 렌더링 최적화에 집중합니다.

## 🎯 트리거 명령어
- 명시적 명령어: **"지식화 하여라"**, **"지식화 해줘"**, **"수정해줘"**
- 자동 발동 조건: 사용자가 새로운 **지식(입력)**이나 **문서**를 제공하거나, 기존 문서의 수정을 요청하는 등 지식 관리가 필요한 **모든 경우**에 발동한다.

---

## 개별 지식 페이지 생성 (Individual Page)
1. **파일 저장 위치 (격리 규칙)**
   - 모든 개별 마크다운(`*.md`) 파일은 해당 카테고리 하위의 **`docs/` 폴더 내부**에 생성한다.

2. **Jekyll Frontmatter (EXCLUDE 필수 반영)**
   - `nav_exclude: true`를 반드시 포함하여 사이드바 중복 노출을 방지한다.
   - **제목 정제**: 제목에서 `[C#]`, `[React]` 등 카테고리 접두사를 제거한다.
   - Frontmatter 예시:
     ---
     layout: post
     title: "{{Dynamic_Emoji}} {{제목}}"
     date: YYYY-MM-DD
     categories: [{{카테고리명}}]
     tags: [{{태그1}}, {{태그2}}]  # 핵심 개념이 없으면 빈칸([]) 유지
     nav_exclude: true
     ---

3. **본문 구조**
   - **Abstract**: 도입부에 `>` 인용구로 본문 핵심을 2~3줄 요약.
   - **Navigation**: 최하단에 `---` 구분선 후 **`[🔙 뒤로 가기](../index.md)`** 링크 필수 삽입.

## 📝 메인 인덱스 업데이트 (index.md)

1. **전체 구조 템플릿 (고정)**
   - 인덱스 파일은 반드시 아래 2단/3단 구조를 유지하며, 각 섹션 사이에는 **반드시 빈 줄**을 둔다.
   - **[상단]**: `# 카테고리명` 및 한 줄 요약
   - **[중단 1]**: `## 📑 참고서적 (선택사항)` (기존 내용 유지)
   - **[중단 2]**: `## 📂 리스트 (총 N건)` (테이블 형식)

2. **표(Table) 작성 엄격 규칙 [CRITICAL]**
   - 표가 시작되기 직전과 끝난 직후에는 **반드시 한 줄 이상의 완전히 비어있는 줄**을 삽입한다.
   - 모든 행의 시작과 끝은 `|` 파이프로 닫는다.
   - `| 제목 | 키워드 |` 아래의 구분선은 `| :--- | :--- |` 형식을 준수한다.

3. **키워드(태그) 관리 규칙**
   - **사전 작성 금지**: 키워드는 에이전트가 임의로 미리 작성하지 않는다.
   - **수정 트리거**: 사용자가 채팅방에 명시적으로 키워드 입력/수정을 요구하는 경우에만 작동한다.
   - **실행 단계**: 요구를 받으면 다음 두 가지를 모두 수정한다.
     1. 해당 문서(`*.md`)의 메타데이터 `tags` 업데이트
     2. 상위 `index.md` 리스트 표(Table)에서 해당 문서 행의 키워드 항목 업데이트
   - 키워드 작성 시 무의미한 단어(카테고리명 중복 등)는 제외하고, 기술적 핵심어만 백틱(`` ` ``)으로 감싼다.

# 🛠️ 문법 (표준 가이드라인)

## 🚨 표 (Table) 렌더링 방어
- **빈 줄 확보**: 표 전후에 텍스트가 붙어있으면 렌더링이 깨진다. 반드시 `\n\n`을 확보하라.
- **특수문자**: 셀 내부에서 `|` 기호를 써야 할 경우 반드시 `\|`로 이스케이프 한다.

## ✨ 이모지 (Dynamic Emoji)
- **맥락 지능**: 이론(🧠), 성능(🚀), 문제해결(🛠️), C#(🎯), JS(💛), AI/에이전트(🤖).

---

## 🤖 [AI 에이전트 지시사항]
1. **표 렌더링 검수**: 출력 전, 표 상하단에 빈 줄이 있는지 다시 확인하라. 이것이 깨지면 가독성이 완전히 무너진다.
2. **중복 제거**: 제목에서 `[카테고리명]`을 절대 쓰지 마라.
3. **참고서적 보존**: 기존 `index.md`에 참고서적이 기록되어 있다면 내용을 지우지 말고 그 아래에 리스트만 업데이트하라.
4. **경로 확인**: 링크는 반드시 `./docs/파일명.md` 구조여야 한다.
5. **즐겨찾기 리스트 관리**: `## ⭐️ 즐겨찾기` 항목을 추가/수정할 때는 표(Table) 형식이 아닌 `- 이모지 [**이름**](./경로)` 형태의 깔끔한 목록(Unordered List)으로 작성 및 유지하라.
6. **키워드 규칙 준수**: 사용자의 명시적 요청이 있기 전까지 키워드는 절대 미리 작성하지 않는다. 요청 시 문서 메타데이터와 인덱스 리스트를 모두 업데이트한다.
