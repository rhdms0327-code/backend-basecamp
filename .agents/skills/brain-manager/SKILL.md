# 🧠 brain-manager: Jekyll 지식 자동화 스킬

## 📌 스킬 개요
사용자가 입력한 지식을 바탕으로 **Jekyll 호환 포스트**를 생성하고, 해당 카테고리의 **메인 인덱스(index.md)**를 실시간으로 구조화합니다. 모든 출력물은 표준 문법 가이드라인을 준수하여 GitHub Pages 렌더링 최적화에 집중합니다.

## 🎯 트리거 명령어
> **"지식화 하여라"** 또는 **"지식화 해줘"**

---

## 📝 [Rule 1] 개별 지식 페이지 생성 (Individual Page)

1. **파일 저장 위치 (격리 규칙)**
   - `index.md`와 섞이지 않도록, 모든 개별 마크다운(`*.md`) 파일은 해당 카테고리 하위의 **`docs/` 폴더 내부**에 생성한다.

2. **Jekyll Frontmatter (EXCLUDE 필수 반영)**
   - 개별 페이지는 좌측 메뉴바에 자동 노출되지 않도록 `nav_exclude: true`를 반드시 포함한다.
   - **태그(tags)**: 한국어 기술 용어를 우선하되, 검색 빈도가 높은 영문 약어는 혼용함. (무의미한 태그는 제외)
   - Frontmatter 작성 예시:
     ---
     layout: post
     title: "{{Dynamic_Emoji}} {{문서 제목}}"  # 카테고리명(예: [C#]) 삽입 금지
     date: YYYY-MM-DD
     categories: [{{카테고리명}}]
     tags: [{{태그1}}, {{태그2}}]  # 추출할 핵심 개념이 없으면 비워둔다. (예: tags: [])
     nav_exclude: true
     ---

3. **본문 구조**
   - **Abstract**: 도입부에 `>` 인용구로 본문 핵심을 2~3줄 요약.
   - **Navigation**: 최하단에 `---` 구분선 후 **`[🔙 뒤로 가기](../index.md)`** 링크 필수 삽입.

## 📝 [Rule 2] 메인 인덱스 업데이트 (index.md)

1. **파일 저장 위치**
   - 인덱스 파일은 개별 파일들과 분리되어, **카테고리 폴더의 최상단(루트)**에 위치한다.

2. **제목 동기화 (Strict Title Matching)**
   - 인덱스 파일(`index.md`)의 `title`과 `#` (H1) 제목은 좌측 메뉴바(사이드바)에 설정된 **카테고리 메뉴명과 100% 일치**하게 작성하라. 임의로 제목을 덧붙이거나 수정하지 마라.

3. **인덱스 테이블 구성 (중복 제거 & Simple)**
   - 카테고리는 폴더 위치상 이미 명확하므로 표 제목에서 `[C#]` 같은 접두사를 과감히 생략한다.
   - **형식 (상대 경로 주의)**:
     ## 📂 리스트 (총 N건)
     
     | 제목 | 키워드 |
     | :--- | :--- |
     | **[{{Dynamic_Emoji}} {{제목}}](./docs/파일명.md)** | `태그1` `태그2` |

4. **키워드(태그) 추출 규칙 (스마트 태깅 & 렌더링 방어)**
   - **무의미한 태그 금지**: 카테고리 이름, 단순 도구 이름, 일반 명사 등 영양가 없는 단어는 태그로 추출하지 마라. 기술적 핵심 개념이 없다면 **키워드 칸을 과감히 비워라.**
   - **한글 우선**: 직관적인 한글 기술 용어를 우선 사용하고, 표준 약어(API, GC 등)는 영문 대문자를 유지한다.
   - **해시태그 절대 금지**: 렌더링 깨짐을 방지하기 위해 **절대 `#` 기호를 붙이지 않는다.** 오직 백틱(`` ` ``)으로만 감싸서 인라인 코드 형식(`가비지컬렉션`)으로 표기한다.

# 🛠️ 문법 (표준 가이드라인)

## 🚨 표 (Table) - [CRITICAL]
- **전후 여백**: 표의 시작 직전과 종료 직후에는 반드시 **최소 한 줄 이상의 빈 줄**을 삽입한다. (렌더링 오류 방지용)
- **파이프 정렬**: 모든 행의 시작과 끝을 `|` 기호로 닫는다.
- **콘텐츠 보호**: 셀 내부의 `|`는 반드시 `\|`로 이스케이프 한다.

## 💻 코드 블록 (Code Block)
- **언어 명시**: 백틱 3개 뒤에 언어(csharp, yaml 등) 명시 필수.
- **주석**: 코드 내 설명은 **한국어**를 우선 사용한다.
- **여백**: 본문과의 구분을 위해 블록 상하단에 빈 줄을 배치한다.

## 💬 콜아웃 (Callouts)
- **GitHub 스타일**: `> [!NOTE]`, `> [!TIP]`, `> [!WARNING]` 형식을 적극 활용한다.
- **가독성**: 긴 텍스트 설명보다 콜아웃을 통해 시각적 위계를 확보한다.

## ✨ 이모지 (Dynamic Emoji)
- **맥락 지능**: 내용에 따라 AI가 판단하여 선택한다.
  - 이론/개념: 🧠, 📖 | 성능/최적화: 🚀, ⚡ | 문제 해결: 🛠️, 🔍 | 신규 업데이트: ✨
- **언어 상징**: C#(🎯), Python(🐍), JS(💛), DB(🗄️) 등을 적절히 조합한다.

## 🤖 [AI 에이전트 지시사항]
1. **디렉토리 분리 엄수**: 개별 문서(`*.md`)는 반드시 `docs/` 하위 폴더에 넣고, 링크 연결 시 `./docs/파일명.md` 구조를 지켜라.
2. **문법 최우선**: 모든 출력 전 **# 문법** 섹션, 특히 표의 빈 줄 여부를 반드시 검수하라.
3. **태그 및 접두사 통제**: 표와 문서 제목에서 `[C#]` 같은 카테고리 접두사를 빼서 가독성을 높여라. 또한 무의미한 태그는 생성하지 마라.
4. **사이드바 제외**: 개별 포스트 Frontmatter에 `nav_exclude: true`가 들어갔는지 확인하라.
5. **메뉴명 일치**: `index.md` 갱신 시, 최상단 H1 제목이 사용자가 의도한 메뉴바 제목과 동일한지 확인하라.
6. **태그 렌더링 방어**: 키워드 표기 시 `#` 기호를 절대 넣지 말고, 백틱으로만 감싸라.
