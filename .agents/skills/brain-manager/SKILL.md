# 🧠 brain-manager: Jekyll 지식 자동화 스킬

## 📌 스킬 개요
사용자가 입력한 지식을 바탕으로 Jekyll 호환 개별 포스트를 생성하고, 해당 카테고리의 메인 인덱스(index.md)를 자동으로 갱신하거나 구조화합니다.

## 🎯 트리거 명령어
> **"지식화 하여라"** 또는 **"지식화 해줘"**

## 📝 [Rule 1] 개별 지식 페이지 생성 (Individual Page)
1. **Jekyll Frontmatter**
   - **tags**: 본문에서 핵심 단어를 추출하되, 영문 지상주의를 탈피합니다. 한글 용어가 더 직관적이라면 한글 키워드를 우선 사용합니다.
   - 예: `tags: [가비지컬렉션, 메모리관리, C#]` (O) / `tags: [GarbageCollection, MemoryManagement, CSharp]` (X - 너무 길고 복잡함)

2. **본문 시각화: 지능형 이모지 (Dynamic Emoji)**
   문서의 성격에 따라 AI가 직관적으로 선택합니다.
   - **이론/원리**: 🧠, 📖, 🎓
   - **최적화/속도**: ⚡, 🚀, ⏱️
   - **에러/디버깅**: 🛠️, 🐛, 🔍, 🚨
   - **구조/설계**: 🏗️, 📐, 🧱
   - **꿀팁/참고**: ✨, 💡, 📎
   - **언어별 상징**: C#(🎯), Python(🐍), JS(💛), DB(🗄️) 등 (문맥에 따라 조합 가능)

3. **본문 구조**
   - **Abstract**: 시작 직후 `>` 인용구로 2~3줄 요약.
   - **Navigation**: 최하단에 `[🔙 뒤로 가기](./index.md)` 링크 필수.

## 📝 [Rule 2] 메인 인덱스 업데이트 (index.md)
1. **인덱스 테이블 구성**
   - **형식**: `| 📑 핵심 아티클 (Title) | 🏷️ 분류 태그 (Tags) |`
   - **내용**: `[[카테고리] {{Dynamic_Emoji}} {{제목}}](./파일명.md)`

2. **키워드(태그) 추출 규칙 (중요!)**
   - **한글 우선 원칙**: 한국어 기술 블로그로서 읽기 편한 단어를 선택합니다.
   - **자연스러운 혼용**: #인터페이스, #추상화처럼 한글이 편한 건 한글로, #API, #JSON처럼 영문 약어가 표준인 건 영문으로 표기합니다.
   - **형식**: 백틱과 샵 사용 (예: `` ` #메모리 ` `` `` ` #GC ` ``)

## 🤖 [AI 에이전트 지시사항]
- **언어 선택의 유연성**: 키워드 추출 시 "개발자가 구글에 검색할 법한 단어" 혹은 "본문에서 주로 사용한 단어"를 기준으로 한/영을 선택하라. 억지로 영단어로 치환하지 마라.
- **맥락 기반 이모지**: 단순히 기술명에 매몰되지 말고, 이 글이 '삽질 기록'인지 '개념 정리'인지에 따라 이모지를 다채롭게 사용하라.
- **Jekyll 규격**: Frontmatter의 형식(날짜, 레이아웃 등)이 깨지지 않도록 엄격히 관리하라.
- **가독성**: 테이블 내의 제목 링크가 본문의 실제 파일명과 일치하는지 확인하라.

✨ **이번 업데이트의 핵심**
- **키워드 현지화**: #DependencyInjection 대신 #의존성주입이나 #DI를 사용하여 가독성이 좋아졌습니다.
- **이모지 자유도**: 똑같은 C# 글이라도 내용에 따라 🎯, ⚡, 🐛 등으로 다양하게 표현되어 블로그가 더 생동감 있어집니다.
