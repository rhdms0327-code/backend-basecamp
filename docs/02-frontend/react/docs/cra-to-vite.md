> 업계 표준이 CRA(Webpack 기반)에서 Vite(ESM 기반)로 완전히 넘어간 이유를 이해하는 것이 핵심.
> Vite는 번들링 없이 브라우저 Native ESM을 직접 활용해 개발 서버 구동 속도를 압도적으로 개선했다.

---

## 1. CRA (Create React App) — Legacy

| 항목 | 내용 |
| :--- | :--- |
| 정체 | 리액트 팀이 공식 제공했던 프로젝트 생성 도구 |
| 기술 기반 | **Webpack** |
| 문제점 | 프로젝트 규모가 커질수록 빌드 및 HMR 속도가 현저히 느려짐 |
| 현재 상태 | 더 이상 공식 권장되지 않는 **Legacy** 방식 |

---

## 2. Vite (비트) — 현재 표준

| 항목 | 내용 |
| :--- | :--- |
| 정체 | 차세대 프런트엔드 빌드 도구 (프랑스어로 '빠르다') |
| 기술 기반 | 개발 모드: 브라우저 **ESM** 활용 / 빌드 시: **Rollup** 사용 |
| 특징 | 초고속 HMR, 경량 설정, 최신 브라우저 최적화 |
| 현재 상태 | 현대 리액트 개발의 **표준(Standard)** |

---

## 📍 핵심: 왜 지금 ESM(ES Modules)인가?

ESM은 자바스크립트 공식 표준 기구(ECMA International)가 승인한 **공식 모듈 시스템** (`import`/`export` 문법).

- **No-bundling**: 과거처럼 모든 파일을 하나로 합칠 필요 없이, 브라우저가 필요한 파일만 서버에 즉시 요청(Native ESM)
- **Vite와의 관계**: 번들링 공정을 생략하고 브라우저 엔진을 그대로 활용 → 서버 구동 속도 '0초'에 가깝게 개선
- **표준의 힘**: 특정 라이브러리에 의존하지 않아 최신 브라우저와 Node.js 환경 어디서든 일관되게 동작
