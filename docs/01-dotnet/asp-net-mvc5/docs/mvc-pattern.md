> 소프트웨어 디자인 패턴인 MVC(Model-View-Controller)의 역할 분담과 유지보수성을 높이는 '느슨한 결합'의 중요성을 학습합니다.

## 1. MVC 구성 요소의 역할

| 구성 요소 | 역할 |
| :--- | :--- |
| **Model (모델)** | 데이터 처리 및 비즈니스 로직 수행 |
| **View (뷰)** | 사용자에게 보여지는 화면(UI) 표현 |
| **Controller (컨트롤러)** | 사용자 요청을 수신하고 모델과 뷰를 연결 |

## 2. MVC 패턴의 주요 특징

1. **역할 분리 (SoC)**: 화면 코드와 데이터 로직이 분리되어 관리가 용이합니다.
2. **독립성**: 디자인(View)을 변경해도 로직(Model)에 영향을 주지 않습니다.
3. **테스트 용이성**: 각 파트를 독립적으로 검증(단위 테스트)하기 좋습니다.
4. **협업 효율**: 디자이너와 개발자의 작업 영역이 명확히 구분됩니다.

## 3. 느슨한 결합 (Loose Coupling)

느슨한 결합이란 클래스들이 서로의 구체적인 구현에 의존하지 않고, **인터페이스(Interface)**를 통해 소통하는 설계 방식입니다.

- **의존성 역전**: 클래스 내부에서 `new`로 객체를 직접 생성하지 않고 외부에서 주입받습니다.
- **유연성**: 내부 로직이 바뀌어도 인터페이스 규약만 유지되면 시스템 전체가 중단되지 않습니다.
- **확장성**: 구현체를 손쉽게 교체할 수 있어 변화에 강한 구조를 가집니다.
