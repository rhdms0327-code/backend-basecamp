> 기존 Web Forms 방식의 한계를 짚어보고, 왜 현대 웹 개발에서 웹 표준과 REST 아키텍처를 준수하는 MVC가 필수가 되었는지 이해합니다.

## 1. 웹 폼(Web Forms)의 한계

기존의 생산성 중심 방식은 다음과 같은 기술적 부채를 남겼습니다.

- **ViewState 부하**: 상태 유지를 위해 거대한 숨겨진 데이터를 HTML에 포함시켜 페이지가 무거워졌습니다.
- **복잡한 수명 주기**: 수많은 이벤트 단계로 인해 세밀한 제어와 디버깅이 어려웠습니다.
- **강한 결합도**: UI와 로직이 뒤섞여 코드 재사용이나 역할 분리가 거의 불가능했습니다.
- **테스트 불가**: 서버 객체에 강하게 의존하여 로직만 따로 검증하는 단위 테스트가 어려웠습니다.

## 2. 현대 웹 개발의 두 기둥

### 웹 표준 (Web Standards)
- 특정 브라우저나 OS에 종속되지 않는 **HTML5, CSS3, JS 표준**을 준수합니다.
- MVC는 MS 전용 태그 대신 순수 HTML을 생성하여 크롬, 사파리, 모바일 등 모든 환경에서 동일하게 작동합니다.

### REST (Representational State Transfer)
- URL로 자원(Resource)을 식별하고 HTTP 메서드로 행위를 정의하는 아키텍처 스타일입니다.
- **클라이언트-서버 분리**: 서버는 데이터만 던져주고 화면은 클라이언트가 알아서 그리는 구조를 가능하게 합니다.
- **무상태성 (Stateless)**: 서버가 클라이언트의 이전 상태를 기억하지 않아 부하가 적고 확장이 쉽습니다.
