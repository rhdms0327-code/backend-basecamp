# 🎯 [.NET 분석] .NET 용어 정리

> .NET 개발 생태계의 기초가 되는 프레임워크와 라이브러리의 차이점, 그리고 통합된 .NET의 의미 등 필수적인 핵심 용어들을 명확히 정리합니다.

## 1. 프레임워크 (Framework) vs 라이브러리 (Library)
- **프레임워크**: 애플리케이션의 기본 뼈대와 흐름을 제공합니다. 개발자는 그 틀 안에서 로직을 구현하며, 제어의 주도권이 프레임워크에 있습니다 (제어의 역전, IoC).
- **라이브러리**: 특정 기능을 수행하는 도구들의 모음입니다. 개발자가 주도적으로 필요할 때마다 호출하여 사용합니다.

## 2. 웹 애플리케이션 (Web Application)
- 웹 브라우저를 UI로 사용하여 상호작용하는 소프트웨어입니다.
- **주요 구성**: 사용자 환경(Client), 서비스 로직(Server), 데이터 저장소(Database)가 유기적으로 연결되어 동작합니다.

## 3. .NET의 두 가지 의미
1. **광의적 의미**: Microsoft가 제공하는 모든 개발 도구, 런타임, 언어를 포함하는 거대한 기술 생태계를 통칭합니다.
2. **협의적 의미**: .NET Framework와 .NET Core의 복잡성을 제거하고 하나로 합쳐진 최신 통합 플랫폼(.NET 5/6/7/8+)을 뜻합니다.

> [!TIP]
> 기술 문서나 커뮤니티에서 단순히 ".NET"이라고 언급한다면, 문맥에 따라 최신 통합 버전(5.0 이후)을 의미하는 경우가 많으므로 주의 깊게 살펴봐야 합니다.
