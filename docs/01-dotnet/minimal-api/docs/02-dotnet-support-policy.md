> .NET 및 .NET Core의 공식 지원 정책 (LTS vs STS), 릴리스 주기 및 수명 종료(EOL)에 대한 학습 내용 정리.

이 문서는 Microsoft의 공식 .NET 지원 정책을 정리한 내용입니다. 프로젝트의 프레임워크 버전을 선택할 때 중요한 기준이 됩니다.

## 1. 릴리스 주기 (Release Cadence)
- .NET의 새로운 주요 릴리스는 **매년 11월**에 발표됩니다.
- **짝수 버전 (예: .NET 6, 8, 10):** LTS (장기 지원) 릴리스입니다.
- **홀수 버전 (예: .NET 5, 7, 9):** STS (표준 기간 지원) 릴리스입니다.

### 📌 지원되는 버전 (.NET & .NET Core)

다음 표는 현재 지원되는 .NET 버전의 릴리스 및 지원 종료 날짜를 나타냅니다.

| 버전 | 릴리스 유형 | 원래 릴리스 날짜 | 지원 종료(EOL) |
| :--- | :--- | :--- | :--- |
| **.NET 10** | LTS (장기 지원) | 2025년 11월 11일 | 2028년 11월 14일 |
| **.NET 9** | STS (표준 기간 지원) | 2024년 11월 12일 | 2026년 11월 10일 |
| **.NET 8** | LTS (장기 지원) | 2023년 11월 14일 | 2026년 11월 10일 |

> [!NOTE]
> .NET 8과 .NET 9의 지원 종료일이 동일하므로 주의가 필요합니다. 2026년 11월 10일 이전에 상위 버전(.NET 10 이상)으로의 업그레이드를 권장합니다.

## 2. 릴리스 유형 및 지원 기간 (LTS vs STS)
두 릴리스의 품질과 기능은 동일하며, 유일한 차이점은 **지원 기간**입니다.

### 📊 LTS vs STS 비교 요약

| 구분 | LTS (장기 지원) | STS (표준 기간 지원) |
| :--- | :--- | :--- |
| **출시 주기** | 짝수 버전 (2년마다) | 홀수 버전 (매년 11월) |
| **지원 기간** | 출시 후 **3년** | 출시 후 **2년** |
| **성격** | 장기 안정성 중심 | 최신 기능 도입 중심 |
| **추천 대상** | 유지보수가 중요한 기업용 시스템 | 신기술 도입이 빠른 프로젝트 |

## 3. 세부 지원 단계

지원 기간 내에서도 시기에 따라 제공되는 업데이트의 성격이 다릅니다.

1. **적극적인 지원 (Active Support):** 
   - 새로운 기능 향상, 심각한 성능 문제 수정, 보고된 버그 패치 및 보안 취약점 완화 등이 포함됩니다.
2. **유지 관리 지원 (Maintenance Support):**
   - 모든 릴리스(STS/LTS) 지원의 **마지막 6개월**에 해당합니다.
   - 주로 보안 취약성 완화 위주의 업데이트만 제공됩니다.
3. **수명 종료 (EOL, End of Life):**
   - 유지 관리 기간이 끝나면 지원이 완전히 종료됩니다.
   - 더 이상 버그 수정이나 보안 업데이트가 제공되지 않으므로, 이 시기 이전에 반드시 최신 버전으로 마이그레이션해야 합니다.
   - EOL 날짜는 보통 해당 월의 "패치 화요일(매월 두 번째 화요일)"에 맞춰 조정됩니다.

## 4. 지원 유지 조건
- LTS 또는 STS에 관계없이, 지원되는 상태를 유지하려면 Microsoft에서 제공하는 최신 패치 업데이트(예: 8.0.x가 나오면 해당 패치 버전 설치)를 지속적으로 적용해야 합니다.
- 패치 업데이트는 누적형이며, 일반적으로 매월 두 번째 화요일(Patch Tuesday)에 릴리스됩니다.
