# [CLR 심층 분석] CLR 관련 개념 정리

## 1. 관리 코드 (Managed Code)
- **런타임(CLR)**에 의해 실행이 관리되는 코드입니다.
- 런타임은 **자동 메모리 관리, 보안, 형식 안전성**과 같은 서비스를 제공합니다.
- C#은 기본적으로 관리 코드를 작성하는 언어입니다.

## 2. 관리되지 않는 코드 (Unmanaged Code)
- C/C++로 작성된 코드처럼 프로그래머가 직접 메모리를 관리(포인터 사용 등)해야 하는 코드입니다.
- C#에서도 `unsafe` 키워드를 사용하여 관리되지 않는 코드를 작성할 수 있습니다.
- 이는 런타임이 제공하는 서비스를 이용하지 않고 프로그래머가 직접 관리하겠다는 의미입니다.

## 3. unsafe 키워드
현대 C# 개발에서 `unsafe`를 직접 쓰는 경우는 드물지만, 다음과 같은 상황에서는 필수적입니다.
- **성능 극대화**: 배열의 인덱스 검사(Boundary Check) 비용마저 아껴야 하는 고성능 이미지 처리나 게임 엔진 개발 시.
- **외부 DLL 호출**: C/C++로 작성된 라이브러리와 데이터를 주고받을 때 주소 값을 넘겨줘야 하는 경우.

```csharp
unsafe 
{ 
    int var = 20; 
    int* ptr = &var; // 변수의 주소를 직접 추출 (포인터) 
    Console.WriteLine($"값: {*ptr}"); // 20 
    Console.WriteLine($"주소: {(long)ptr}"); // 실제 메모리 주소 출력 
}
```

## 4. 중간 언어 (Intermediate Language, IL)
- 빌드 결과물인 `.dll`, `.exe` 파일은 기계어가 아니라 **IL**로 만들어져 있습니다.
- **언어 통합**: C#, VB, F# 등 어떤 언어로 작성하든 동일한 IL로 변환되어 언어 간 라이브러리 공유가 가능합니다.
- **독립성**: 특정 CPU에 종속되지 않아 JIT 컴파일러만 있다면 어디서든 실행 가능합니다.
- **흐름**: 소스 코드 → IL (빌드) → 기계어 (실행 시 JIT)

## 5. CLR 동작 과정
1. **컴파일 타임 (빌드 시)**: C# 컴파일러가 소스 코드를 읽어 중간 언어(IL)로 변환하고 메타데이터와 함께 `.dll` 또는 `.exe`에 저장합니다.
2. **런타임 (실행 시)**:
   - **로딩**: CLR이 IL을 읽어 `Loader Heap`에 타입 설계도(`MethodTable`)를 생성합니다.
   - **JIT 컴파일**: 메서드가 처음 호출될 때 JIT 컴파일러가 IL을 기계어로 실시간 번역합니다.
   - **캐싱**: 번역된 기계어는 `JIT Code Heap`에 저장되어 재사용됩니다.
   - **실행**: 스택과 힙을 사용하여 로직을 수행합니다.
   - **관리**: 사용되지 않는 객체는 **GC(가비지 컬렉터)**가 자동으로 회수합니다.

## 6. JIT 컴파일 과정의 장점
- **지연 번역 (Lazy Compilation)**: 실제로 호출된 메서드만 번역하여 메모리와 CPU 리소스를 절약합니다.
- **캐싱**: 한 번 번역된 코드는 다시 번역할 필요 없이 네이티브 속도로 실행됩니다.
- **플랫폼 독립성**: 빌드 시에는 IL만 생성하므로 하나의 파일로 여러 환경(Windows, Linux 등)에서 실행 가능합니다.
- **실행 시점 최적화**: JIT가 실행 중인 CPU의 특수 명령어를 파악하여 최적화된 기계어를 생성합니다.
