# 🎯 [8장] 인터페이스 (Interface)

> 클래스가 외부와 소통하는 규약인 인터페이스의 개념을 파악하고, C# 8.0의 기본 구현 메서드 및 추상 클래스와의 차이점을 정리합니다.

## 📖 1. 개요
- 클래스가 구현해야 하는 기능의 **계약(Contract)**을 정의합니다.
- 메서드, 프로퍼티, 이벤트의 시그니처만 포함하며, 실제 구현은 이를 상속받은 클래스에서 책임집니다.

## 2. 기본 구현 메서드 (Default Implementation)
C# 8.0부터 인터페이스 내부에 메서드의 기본 실행 코드를 작성할 수 있게 되었습니다.
- **하위 호환성**: 기존 인터페이스에 새로운 기능을 추가할 때, 이미 인터페이스를 구현하고 있는 수많은 클래스를 수정하지 않아도 됩니다.
- **유연성**: 클래스에서 필요한 경우에만 선택적으로 메서드를 재정의할 수 있습니다.

## 3. 인터페이스 vs 추상 클래스
- **인터페이스**: '무엇을 할 수 있는가'(Behavior)에 집중하며, 클래스당 여러 개를 다중 구현할 수 있습니다.
- **추상 클래스**: '무엇인가'(Identity)에 집중하며, 상태(필드)를 가질 수 있고 단일 상속만 허용됩니다.

> [!TIP]
> "비행기"와 "새"는 서로 다른 종(추상 클래스)이지만, 둘 다 "날 수 있다(IFlyable)"는 공통 기능(인터페이스)을 가질 수 있습니다. 이처럼 공통 행위를 묶을 때 인터페이스를 활용하세요.

## 4. 활용 가치
- **느슨한 결합(Loose Coupling)**: 구체적인 클래스 대신 인터페이스에 의존하게 설계하여 부품을 교체하듯 코드를 쉽게 변경할 수 있습니다.
- **다형성 구현**: 서로 다른 객체들을 동일한 인터페이스 타입으로 관리하여 일관된 로직을 적용할 수 있습니다.
