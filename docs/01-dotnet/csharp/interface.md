# [8장] 인터페이스 (Interface)

## 1. 개요
- 클래스가 구현해야 하는 **계약(Contract)**을 정의합니다.
- 메서드, 프로퍼티, 이벤트 등의 선언만 포함하며, 실제 구현은 클래스에서 담당합니다.

## 2. 기본 구현 메서드 (Default Implementation)
C# 8.0부터 인터페이스 내부에도 메서드의 본문을 직접 정의할 수 있게 되었습니다.
- **장점**: 기존 인터페이스에 새로운 메서드를 추가할 때, 상속받은 수많은 클래스를 일일이 수정하지 않아도 됩니다.
- **특징**: 클래스에서 재정의(`override`)하지 않으면 인터페이스의 기본 구현이 사용됩니다.

## 3. 인터페이스 vs 추상 클래스
- **인터페이스**: '할 수 있는 기능' (Can-do) 정의. 다중 상속 가능.
- **추상 클래스**: '무엇인가' (Is-a) 정의. 단일 상속만 가능. 상태(필드)를 가질 수 있음.

## 4. 활용
- **다형성**: 다양한 클래스를 동일한 인터페이스 타입으로 묶어 처리할 수 있습니다.
- **느슨한 결합**: 구체적인 구현 클래스 대신 인터페이스에 의존하게 하여 유지보수성을 높입니다.
