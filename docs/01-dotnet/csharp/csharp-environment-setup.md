# 🎯 [1장] C# 파일 구조 및 환경 설정

> C# 개발을 시작하기 위한 첫 번째 단계로, 프로젝트 구조의 핵심이 되는 파일 확장자와 컴파일 흐름, 그리고 개발 환경 구성 방법을 정리합니다.

## 1. 주요 용어 및 확장자
- **.cs**: C# 소스 코드가 작성된 파일입니다.
- **.csproj**: 프로젝트 설정, 사용되는 SDK 버전, NuGet 패키지 참조 정보를 담고 있는 XML 기반 설정 파일입니다.
- **.sln**: 솔루션 파일로, 연관된 여러 개의 프로젝트를 하나의 작업 단위로 묶어 관리합니다.

## 2. 컴파일 과정
1. **소스 코드**: 개발자가 작성한 `.cs` 파일입니다.
2. **컴파일**: C# 컴파일러가 소스 코드를 **IL(중간 언어)**로 변환하여 `.dll` 또는 `.exe` 파일을 생성합니다.
3. **실행**: 프로그램을 실행하면 **CLR**이 IL을 실시간으로 기계어로 번역(JIT 컴파일)하며 실행합니다.

## 🛠️ 3. 환경 설정
- **Visual Studio**: .NET 개발에 최적화된 IDE입니다. 설치 시 'C# 및 .NET 데스크톱 개발' 워크로드를 반드시 포함해야 합니다.
- **SDK**: .NET 앱을 빌드하고 실행하는 데 필요한 라이브러리와 도구 모음입니다.

> [!TIP]
> 경량화된 환경을 선호한다면 **VS Code**에 **C# Dev Kit** 확장 프로그램을 설치하여 동일한 SDK 기반의 개발 환경을 구축할 수 있습니다.
