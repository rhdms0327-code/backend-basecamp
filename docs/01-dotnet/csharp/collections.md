# 🎯 [10장] 자료구조 (컬렉션 - Collections)

> 데이터를 효율적으로 저장하고 관리하기 위한 .NET의 다양한 컬렉션 클래스들을 살펴보고, 상황에 맞는 자료구조 선택 가이드를 제공합니다.

## 📖 1. 개요
- 데이터를 효율적으로 저장하고 관리하기 위해 제공되는 클래스 라이브러리입니다.
- 고정 크기인 배열과 달리 **동적 크기** 조절이 가능하며 다양한 데이터 관리 방식을 지원합니다.

## 2. 주요 컬렉션 종류
- **List<T>**: 가장 많이 쓰이는 가변 배열로 인덱스 기반 접근이 가능합니다.
- **Dictionary<TKey, TValue>**: 키(Key)와 값(Value)의 쌍으로 저장하며 검색 속도가 매우 빠릅니다.
- **Queue<T>**: 선입선출 (FIFO) 방식의 자료구조입니다.
- **Stack<T>**: 후입선출 (LIFO) 방식의 자료구조입니다.
- **HashSet<T>**: 중복을 허용하지 않는 유일한 요소들의 집합입니다.

## 3. 배열 vs 컬렉션

| 구분 | 배열 (Array) | 컬렉션 (Collection) |
| :--- | :--- | :--- |
| **크기** | 고정 크기 | 가변 크기 (동적 조절) |
| **성능** | 인덱스 접근이 가장 빠름 | 기능에 따라 성능 차이 발생 |
| **유연성** | 낮음 | 높음 (추가, 삭제 용이) |

> [!TIP]
> `ArrayList`와 같은 비일반화 컬렉션은 박싱/언박싱으로 인한 성능 저하가 발생하므로, 현대적인 C# 개발에서는 반드시 `List<T>`와 같은 제네릭(일반화) 컬렉션을 사용해야 합니다.

## 4. 선택 가이드
- 순차적 접근과 빠른 인덱스 조회가 필요하면 `List<T>`.
- 데이터의 유일성이 중요하면 `HashSet<T>`.
- 키 값을 이용한 빠른 조회가 필요하면 `Dictionary<TKey, TValue>`.
