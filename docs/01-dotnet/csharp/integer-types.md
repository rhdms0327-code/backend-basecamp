# 🎯 [3장] 정수 계열 (Integer Type)

> C#에서 제공하는 다양한 크기의 정수 형식들의 범위와 특징을 비교하고, 데이터 크기에 따른 최적의 타입 선택 기준을 정리합니다.

## 1. 정수 형식 표

| 형식 | 크기 | 범위 | 비고 |
| :--- | :--- | :--- | :--- |
| **byte** | 1바이트 | 0 ~ 255 | 부호 없는 8비트 |
| **sbyte** | 1바이트 | -128 ~ 127 | 부호 있는 8비트 |
| **short** | 2바이트 | -32,768 ~ 32,767 | 16비트 |
| **int** | 4바이트 | 약 -21억 ~ 21억 | 32비트 (기본 정수형) |
| **uint** | 4바이트 | 0 ~ 약 42억 | 부호 없는 32비트 |
| **long** | 8바이트 | 약 -922경 ~ 922경 | 64비트 |

## 2. 선택 기준
- **범용성**: 특별한 이유가 없다면 CPU 연산에 가장 효율적인 **int** 사용을 권장합니다.
- **범위 확장**: 인구수, 대용량 파일 크기 등 21억을 초과할 가능성이 있다면 **long**을 선택합니다.
- **최적화**: 수백만 개의 숫자를 담는 배열이나 네트워크 패킷 처리 시에는 **byte**나 **short**로 메모리를 절약합니다.

## 3. 오버플로 (Overflow)
- 변수의 최대 범위를 초과하는 연산 시 비트가 넘쳐 최소값으로 회전하는 현상입니다.
- **안전 장치**: `checked` 블록을 사용하면 오버플로 발생 시 런타임 예외를 발생시켜 데이터 무결성을 보호할 수 있습니다.

> [!WARNING]
> 기본적으로 C#은 성능을 위해 오버플로 검사를 하지 않습니다. 중요한 금융 연산 등에서는 반드시 `checked`를 활용하거나 프로젝트 설정에서 체크 옵션을 활성화해야 합니다.
