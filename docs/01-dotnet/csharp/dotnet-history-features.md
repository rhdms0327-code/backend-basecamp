# 🎯 [.NET 분석] .NET의 역사와 주요 특징

> 2002년부터 시작된 .NET Framework의 탄생부터, 오픈 소스 기반의 .NET Core를 거쳐 현대의 통합된 .NET에 이르기까지의 진화 과정과 핵심 가치를 조명합니다.

## 1. .NET Framework (2002 ~ 2019)
- **시작**: Microsoft가 Java에 대응하여 출시한 Windows 전용 개발 플랫폼입니다.
- **특징**: 폐쇄형 소스였으며, Windows 운영체제와 밀접하게 결합되어 강력한 데스크톱 앱 생태계를 구축했습니다.
- **현재**: 4.8.x 버전을 마지막으로 기능 추가는 중단되었으며, 현재는 안정적인 유지보수 단계에 있습니다.

## 2. .NET Core (2016 ~ 2019)
- **등장 배경**: 클라우드 네이티브 환경과 크로스 플랫폼(Linux, macOS) 지원을 위해 완전히 새롭게 재설계되었습니다.
- **특징**: 오픈 소스로 전환되었으며, 필요한 라이브러리만 선택적으로 사용하는 모듈화된 설계를 통해 성능을 극대화했습니다.

## 3. 통합된 .NET (2020 ~ 현재)
- **.NET 5/6/7/8**: 파편화된 명칭을 정리하고 단일 플랫폼으로 통합했습니다.
- **최신 상태**: 현재는 **.NET 8 (LTS)** 버전이 주력이며, 하나의 런타임으로 모든 종류의 애플리케이션을 개발할 수 있는 비전을 실현 중입니다.

> [!TIP]
> 실무에서 프로젝트 버전을 선택할 때는 장기 지원 버전인 **LTS(Long Term Support)**를 선택하는 것이 안정적인 서비스 운영에 유리합니다. (.NET 6, .NET 8 등)

## 4. .NET의 핵심 가치
- **오픈 소스**: GitHub을 통해 전 세계 개발자들과 소통하며 발전합니다.
- **크로스 플랫폼**: 단일 코드베이스로 모든 주요 OS 및 모바일 환경을 지원합니다.
- **고성능**: 동급 대비 최상의 실행 성능과 메모리 효율을 자랑합니다.
